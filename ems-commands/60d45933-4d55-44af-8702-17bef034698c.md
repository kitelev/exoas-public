# Create ActionPrototype instance

Grounding for the "Create Action Instance" command (`169fa2ab`), bound to
`ems__ActionPrototype` layouts. Sibling of "Create TaskPrototype instance"
(`a6ef8fda`) — instantiates an `ems__Action` from the clicked prototype.

- `targetClass` → `ems__Action` (`6a99d2ca`).
- The new instance's `exo__Asset_prototype` back-link to the clicked prototype is
  written by the Universal IR `cf1ed702` (`_targetClassCondition: ems__ActionPrototype`)
  attached to the Universal Default Template singleton — same mechanism as TaskPrototype.
- `prefillLabelWithDate` + `labelTemplate` → instance label like
  «Выпить Пантокальцин 2026-06-29» (dated daily-intake log).
- PropertyDefault `d9aa9bb8`: starts as `ems__EffortStatusDraft`; user taps
  "Execute" to mark it Done. No `ems__Recurring` / auto-generation (VL2).
