# ems__Effort_prevIteration

- **Domain**: [[086f71fa-dd30-4284-90cf-e609f2a6c461]] (declared on superclass per §4.2 sibling-level placement, not on ems__Task subclass)
- **Range**: [[086f71fa-dd30-4284-90cf-e609f2a6c461]]
- **Cardinality**: 0..1 (functional — each effort has at most one prior iteration)
- **owl:equivalentProperty**: `prov:wasRevisionOf` — bridge to W3C PROV-O ontology for external tools (e.g., provenance trackers, lineage explorers)

## Description
- Previous iteration in a recurring effort chain
	- Links current effort instance to the immediately preceding one it supersedes/revises
	- Domain placed on ems__Effort superclass (not ems__Task) so the property is reusable for ems__Project, ems__Meeting, and any future Effort subclass
- **Inverse derivable via SPARQL** — no explicit `nextIteration` property needed
	- Query pattern: `?next ems:Effort_prevIteration ?this`
	- Aligns with UWI invariant (child → parent only); inverse views resolved at query time
- **PROV-O bridge** via `owl:equivalentProperty prov:wasRevisionOf`
	- Enables external semantic tooling to consume the chain without Exocortex-specific vocabulary
