# ems__WorkflowTransition_postActions

Object property. Domain: `ems__WorkflowTransition`. Range: any
`exocmd__Grounding` (typically composite / property_set / property_delete).
Cardinality: 0..N (transition without post-actions = pure status change).

## Execution semantics

`GroundingExecutor.executeWorkflowTransition` runs postActions sequentially
after status mutation. On any postAction failure → halt and return error
(matches `executeComposite` semantics).

## Order

YAML frontmatter array order is preserved. If finer control needed, use
`exocmd__Grounding_order` (future Phase 3).
