# Create Task for DailyNote → pn__DailyNote binding

Binds the **«Create Task for DailyNote»** command ([[c6b64b55-460b-4073-aa9f-538628b678e3]]) to assets of class `pn__DailyNote`. Renders the inline button in the **creation** group on daily-note layout (order=245, between «Create Task» 240 и siblings 250+).

Завершает homoiconic-цепочку для Grounding [[4d8d5055-cf83-4d12-8749-7c0f6956d9be]]:

| Слой | Ассет |
|---|---|
| Grounding (что создаём) | [[4d8d5055-cf83-4d12-8749-7c0f6956d9be]] |
| Command (UI metadata) | [[c6b64b55-460b-4073-aa9f-538628b678e3]] |
| **Binding (где показывать) — этот ассет** | self |

## Восстанавливаемая регрессия

CommandBinding отсутствовал до 2026-05-23 — кнопка «Create task for daily note (declarative)» не рендерилась на daily notes несмотря на existing Grounding + Command. Diagnosed в orchestrator session `e77ae258-0bc6-4568-b33f-4164633ab03c` после Phase 4a/4b shipped + plugin upgrade до v16.19.2, когда стало ясно что Grounding+Command — корректны, а недостающее звено — именно binding.

Не связано с RFC v2 migration phases (3b/3.5/4a/4b/5) — pre-existing gap.
