# Create task for daily note (declarative)

Creates an `ems__Task` in the same folder as the host `period__DailyNote` and
seeds `ems__Effort_plannedStartTimestamp` to today (`YYYY-MM-DD`). Folder
resolution uses the new `$targetFolder` substitution token; the planned-start
seed uses `propertyDefaults` so userInput can still override it.

Issue #3136 (RFC `18407cb2-9554-4897-9213-17321f9dd434` Path B, Q3.b closure)
— migrated from `service_call` + `serviceId: createTaskForDailyNote` to
declarative `create_instance` once the `$targetFolder` substitution token and
`exocmd__Grounding_propertyDefaults` JSON literal landed in the engine.
Behaviorally identical to the previous service call (folder, class, planned
start), with the back-link to the host daily note written via the standard
`create_instance` linkBack mechanism.

**RFC v2 Phase 4a (Issue #3165, 2026-05-23):** the legacy JSON-literal
`exocmd__Grounding_propertyDefaults: '{"ems__Effort_plannedStartTimestamp":"$today"}'`
was replaced with a ref-form `exocmd__Grounding_propertyDefault` pointing to a
new `exocmd__PropertyDefault` instance (`c4e16484-6d19-4406-b57d-19000b7d1560`)
which itself references the canonical `$today` `exocmd__SubstitutionToken`
instance (`665ab8ac-8b9b-472d-9c0a-f82b9abf11bd`). This eliminates the last
opaque JSON literal in this Grounding's frontmatter and makes the value-source
discoverable via SPARQL (RFC v2 §SubstitutionToken).
